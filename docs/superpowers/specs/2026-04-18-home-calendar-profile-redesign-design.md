# CPA 小程序首页/日历/我的重构设计稿

## 背景

当前小程序首页文字密度过高，说明性卡片过多，主功能不够聚焦。用户希望页面更简洁，突出 `开始学习`、`学习热力反馈` 和 `个人学习成果`，并把底部主导航收缩为 3 个 tab。

本次重构只调整前端信息架构、视觉层级和少量统计展示，不改变现有核心业务规则：

- 学习会话仍然通过开始/暂停/恢复/结束完成
- 结束学习后仍需拍照并填写一句话总结
- 热力图仍按每日总学习时长加深颜色
- 共享主页功能保留，但不再占用底部 tab

## 目标

1. 首页减少文字使用，突出计时器和热力图。
2. 底部 tab 从 4 个收缩为 3 个：`打卡 / 日历 / 我的`。
3. 日历页聚焦完整热力图和单日学习情况。
4. “我的”页从资料页改为个人学习总览页。
5. 共享主页入口并入“我的”页作为二级入口。

## 非目标

- 不改动后端会话规则和打卡提交流程
- 不新增提醒、广场、社交互动
- 不改动公开主页本身的主展示结构
- 不引入新的底层状态管理方案

## 最终信息架构

### Tab 结构

- `打卡`
- `日历`
- `我的`

### 二级页面

- `结束打卡页`
- `日详情页`
- `公开主页页`
- `首次资料完善页`
- `共享主页管理页`

共享页从底部 tab 中移除，但其页面和能力继续保留，通过“我的”页进入。

## 页面设计

### 1. 打卡页

#### 页面目标

让用户打开首页后立刻进入学习状态，不需要先阅读说明。

#### 页面结构

1. `欢迎语卡片`
2. `大计时器卡片`
3. `当月缩略热力图`

#### 欢迎语卡片

- 显示每日一句中英双语励志短句
- 保持 2 行结构：
  - 英文主句
  - 中文副句
- 视觉上是轻量卡片，不占据过多垂直空间

示例：

- `One page at a time.`
- `一页一页，也是在前进。`

#### 计时器卡片

这是首页唯一主视觉区域。

状态只保留短标签：

- `准备开始`
- `专注中`
- `已暂停`

交互规则：

- 未开始：显示单主按钮 `开始`
- 进行中：显示 `暂停 / 结束`
- 已暂停：显示 `继续 / 结束`

不再放长段解释文案，只显示：

- 当前状态
- 已学习时长
- 操作按钮

#### 当月缩略热力图

- 显示当前月份的缩略热力月历
- 右上角显示本月总学习时长
- 点击任意日期跳转到日历页对应日期详情
- 不额外放“颜色说明段落”，只通过颜色深浅传达强度

#### 页面原则

- 首页不出现大段说明文字
- 不重复展示“最近总结”“累计进度”等次级信息卡片
- 视觉层级固定为：
  - 计时器
  - 热力图
  - 欢迎语

### 2. 日历页

#### 页面目标

作为完整热力图主场，集中展示本月学习情况和某一天的学习明细。

#### 页面结构

1. 顶部月份栏
2. 完整月热力图
3. 当日详情区

#### 顶部月份栏

- 当前月份
- 本月总学习时长
- 上月 / 下月切换

#### 热力图

- 沿用当前热力等级映射规则
- 颜色从极浅薄荷到深绿色递进
- 点击日期后，下方展示该日详情

#### 当日详情区

显示：

- 当日总学习时长
- 当日学习次数
- 每次学习总结
- 每次学习照片缩略图
- 科目与标签

页面重点是“看每天学了多少、学了什么”，因此不再堆砌额外介绍文案。

### 3. 我的页

#### 页面目标

把当前资料页改造成“个人学习总览页”。

#### 页面结构

1. 昵称信息区
2. 科目总学习时长
3. 最长学习日
4. 累计总学习时长
5. 二级入口区

#### 昵称信息区

- 展示头像
- 展示昵称

#### 科目总学习时长

按固定 CPA 科目汇总展示：

- 会计
- 审计
- 税法
- 财管
- 经济法
- 战略

表现形式为简洁排行列表或条形进度项，不使用大段文字解释。

#### 最长学习日

展示：

- 哪一天学习时长最长
- 该日总时长

#### 累计总学习时长

- 单独做数字卡片展示

#### 二级入口区

保留两个入口：

- `共享主页管理`
- `编辑资料`

“共享主页管理”替代原来的底部“共享”tab。

## 文案策略

### 保留文案

- 每日一句欢迎语
- 极短状态词
- 极短按钮文案
- 极短统计标签

### 删除文案

- 长段功能说明
- 首页解释性总结文案
- 重复性的指导性文字

### 文案风格

- 简短
- 轻量
- 不说教
- 以数字和图形为主，文字为辅

## 视觉风格

### 关键词

- 薄荷学习风
- 轻盈
- 干净
- 大圆角
- 少层级

### 颜色

- 主色：薄荷绿
- 背景：奶白 + 很浅的青绿色过渡
- 高亮：深一层绿色，用于主按钮和高热力格

### 布局原则

- 减少“多层卡片套卡片”
- 保留留白
- 让一个页面只突出一个主视觉

### 首页视觉顺序

1. 计时器
2. 热力图
3. 欢迎语

## 数据与接口补充

本次重构不改动核心会话接口，但“我的”页需要新增或补充聚合统计能力。

### 需要展示的新统计

1. 各科总学习时长
2. 学习时长最长的日期及对应总时长

### 建议实现方式

优先在现有首页/分享类聚合接口之外新增一个面向“我的”页的聚合响应，避免前端自行拼接多次请求。

建议响应包含：

- profile
- totalMinutes
- subjectTotals
- bestDay
- currentStreakDays
- share settings summary

其中：

- `subjectTotals` 为按科目聚合后的分钟数
- `bestDay` 为学习时长最长的自然日统计

## 兼容与迁移

### 需要改动

- `app.json` tab 配置
- 自定义 tabBar 组件
- 首页 WXML / WXSS / TS
- 日历页 WXML / WXSS / TS
- 我的页 WXML / WXSS / TS
- 我的页聚合接口或前端数据组装逻辑

### 保留页面

- `pages/share` 从 tab 中移除，但页面文件保留
- 分享逻辑继续通过二级入口访问

## 风险与约束

### 风险 1：我的页统计不足

如果当前后端没有按科目汇总和最长学习日统计，则“我的”页需要额外接口支持或在服务端补聚合。

### 风险 2：首页信息删减过度

如果首页只剩极少文字，欢迎语和统计层级必须控制好，否则会显得空。

### 风险 3：日历页与首页热力图重复

首页必须用“缩略概览”，日历页负责“完整查看”，不能做成两个相似页面。

## 验收标准

1. 底部 tab 仅保留 `打卡 / 日历 / 我的`
2. 首页仅突出欢迎语、计时器和当月缩略热力图
3. 首页不再出现大段说明文字
4. 日历页可以清晰查看每一天学习情况
5. 我的页可以看到：
   - 昵称
   - 各科总学习时长
   - 最长学习日
   - 累计总学习时长
   - 共享主页管理入口
6. 原有打卡、拍照、总结和热力图规则不受影响

# 小猫专注 · UI 重设计提示词

> 给 Claude Design / Figma Make / v0 等 AI 设计工具的完整设计简报。可直接复制粘贴。

## 1. 项目概述

帮我为「**小猫专注**」(WeChat 微信小程序) 重新设计完整 UI 系统。这是一款面向中国 CPA (注册会计师) 备考人群的**专注计时 + 学习打卡**工具，用户在备考周期 (6-18 个月) 中每天使用 1-3 小时。设计目标是支撑长期使用而不引起视觉疲劳，并强化"专注"这件事本身的仪式感。

## 2. 目标用户

- 25-35 岁，CPA 备考中，多为在职考生 (周末 + 晚上学习)
- 高度自驱，对花哨游戏化反感，希望工具不打扰
- 普遍 iPhone 11-15 用户，全面屏 + OLED
- 心理状态：备考焦虑、自我怀疑、需要被"稳定的进度"支撑

## 3. 设计原则 (按优先级排序)

1. **沉浸优先于装饰** — 专注计时是核心场景。session 进行中应该有"世界缩成计时器"的视觉收束感。
2. **信息密度低 > 高** — 每屏不超过 3 个主要信息块。次要信息折叠到二级页或 tooltip。
3. **静默优于反馈** — 默认动效慢、淡、可被忽略。只有"成就解锁"这类罕见事件值得喧哗。
4. **数据驱动美感** — 大字号数字 (累计学时、考试倒计时) 是视觉支点。卡片、图标都是衬托。
5. **不要游戏化感** — 这不是 Duolingo，不是 Forest。用户在备考，是工作不是娱乐。
6. **续航友好** — OLED 上的大面积浅色背景会持续耗电；focus mode 默认深色优先。
7. **iOS 优先，但兼顾 Android 渲染差异** — 字号、阴影、模糊都要在低端 Android 设备验证。

## 4. 反原则 (明确不要做)

- ❌ 不用渐变绿色"积极心理学"配色 (像 Forest 那种)
- ❌ 不用拟物化插画 / 3D 物件 / glassmorphism (毛玻璃)
- ❌ 不用 emoji 装饰文案 (🎉🌟✨这类)
- ❌ 不用励志口号 ("继续努力！" / "坚持就是胜利！")
- ❌ 不用奖励性弹窗 (除非真正的成就解锁瞬间)
- ❌ 不用过度动效 (>500ms 的转场、parallax、视差)

## 5. 视觉系统

### 调色板

主色:
- **深墨绿 #155946** — 主品牌色，用于大字号数字、关键 CTA
- **薄荷绿 #2EA985** — 次要强调色、链接、激活状态
- **奶绿 #F3FAF6** — 浅模式背景
- **碳灰 #1A1F1C** — 深模式背景 (focus mode 首选)

中性灰阶 (使用 OKLCH 色域而非 HSL):
- 文字主 #1F2624
- 文字次 #5E7D75
- 描边 #E3EBE7
- 占位 #C2D0CB

稀有度配色 (用于成就系统，仅在那一处使用):
- 普通: 雾灰 #98ABA4
- 稀有: 雾蓝 #4A6ED9
- 史诗: 琥珀 #E57B2B
- 传说: 金沙 #C9881C

警告 / 错误 / 成功 — 与主色保持距离:
- 警告: 砖橙 #C2562B
- 错误: 暗红 #A0322B
- 成功: 同主色，不另立色

### 字体

- 中文: PingFang SC (iOS) / 系统默认 sans (Android)
- 数字: SF Pro Rounded Tabular (tabular-nums 启用)
- 不使用衬线字体
- 不使用斜体 (中文斜体几乎都是机械变形，难看)

字号阶梯 (rpx，750 设计宽度):
- Display: 128 (focus mode 计时器，唯一)
- Hero: 64 (考试倒计时数字、成就解锁名)
- Title-1: 40 (页面标题)
- Title-2: 32 (卡片标题)
- Body: 28 (正文)
- Caption: 22 (说明)
- Micro: 18 (极次要标签)

字重: 仅用 400 / 600 / 800 三档。不用 500/700 (视觉差异不够明显，浪费字重位)。

### 间距

8-base 网格:
- 紧密: 8rpx
- 默认: 16rpx
- 段落: 24rpx
- 卡片间: 40rpx
- 页边距: 32rpx

### 圆角

仅用 4 档:
- 小 (chip / pill): 999rpx (全圆)
- 中 (按钮 / 输入框): 16rpx
- 大 (卡片): 24rpx
- 超大 (modal / 弹窗): 32rpx

不要混用其他圆角值。

### 阴影

仅用 3 档:
- 浮起 (卡片): `0 2rpx 8rpx rgba(20, 60, 50, 0.04)`
- 突出 (浮层): `0 12rpx 32rpx -4rpx rgba(20, 60, 50, 0.08)`
- 强调 (modal): `0 24rpx 60rpx -8rpx rgba(20, 60, 50, 0.16)`

无内阴影。无彩色阴影。

### 动效曲线

仅用 2 条曲线:
- 标准: `cubic-bezier(0.2, 0.8, 0.3, 1)` — 进入、值变化
- 退出: `cubic-bezier(0.4, 0, 1, 1)` — 关闭、移除

时长档:
- 100ms — 按压反馈、tooltip
- 220ms — 状态切换 (chip 选中)
- 320ms — 页面转场、modal 进退
- 600ms — 庆祝动画 (成就解锁专属)

呼吸动画 (focus mode 计时器): 4s 完整周期，`opacity 1 → 0.86 → 1` + `scale 1 → 0.985 → 1`，`ease-in-out`。

## 6. 组件清单 (含所有状态)

### 6.1 Button

四种:
- **Primary CTA** — 深墨绿背景，白字，主操作
- **Ghost** — 透明背景，深墨绿边 + 深墨绿字，次要操作
- **Danger** — 砖橙边 + 砖橙字 (限删除/重置)
- **Link inline** — 仅文字，下划线只在 hover (无下划线默认)

状态: default / pressed (scale 0.97) / disabled (opacity 0.55) / loading (内嵌 spinner 居中)

⚠️ **WeChat `<button>` 默认 `::after` 描边必须重置为 `border: none !important`，否则文字会微偏。**

### 6.2 Card

- 默认: 白底 / 深模式 #2A2F2D，圆角 24rpx，浮起阴影
- Hero card: 内置一个 hero 数字 (Hero size) + 一行说明 (Caption)
- 可点击 card: hover/pressed 时整卡 scale 0.99

### 6.3 Chip / Pill

- 默认: 浅奶绿 #ECF5F0 / 深 #2D3433 背景，深字
- 激活: 主品牌色背景，白字
- 单选行: 一组只允许一个激活，激活时其余视觉权重弱化
- 多选行: 激活与未激活区分但都保持饱和度

混合 emoji + 中文 + 数字时 **必须** 用 `display: inline-flex; align-items: center; gap` 拆分子元素，不要塞进单个 `<text>` (emoji 和 CJK baseline 不同，会偏移)。

### 6.4 Stepper (数值步进)

- 左 − / 中数值 / 右 + 三段
- 数值用 tabular-nums，宽度固定避免抖动
- 到达上下限时按钮变 disabled 灰

### 6.5 Toggle Switch

- 自定义实现 (不用原生 `<switch>`)，跟自定义 tab bar 风格统一
- 关: 浅灰胶囊 + 白圆点左
- 开: 深墨绿胶囊 + 白圆点右
- 切换动效: 220ms 标准曲线

### 6.6 Modal / Bottom Sheet

- 优先 Bottom Sheet 而非全屏 Modal (微信小程序惯例)
- 顶部留 32rpx 圆角，从底部滑出 320ms
- 背景遮罩 rgba(20,60,50,0.55)，点击关闭
- 内部内容滚动时遮罩透明度不变

### 6.7 Tab Bar (底部)

- 3 tab: 首页 / 日历 / 我的
- 自定义 component (非原生 tabBar)，悬浮胶囊形状，圆角 999rpx
- 距底 24rpx + iOS safe area
- 激活: 深墨绿背景胶囊，白字
- 未激活: 透明背景，深灰字
- **focus mode 期间整条 tab bar 隐藏** (translateY + opacity 转场)

## 7. 页面架构

### 7.1 首页 — 待机态 (idle)

固定结构，自上而下:
1. 考试倒计时 Hero (Hero 64rpx 数字 + Caption 22rpx 说明)
2. 计时器卡片 (中央 Display 96rpx 数字 + 主 CTA "开始专注")
3. 今日目标 + 挑战 (单卡，目标进度条 + 内联挑战 pill)
4. 六科进度 (横向 6 条 mini bar，可选展开)

**总块数 ≤ 4**。多了就要砍。

### 7.2 首页 — 专注态 (focus mode)

session 激活时整页变身:
- 背景: 深模式纯黑 + 顶部微淡的薄荷弱光 radial-gradient
- 上半区: 极小字 "今日 X 分钟 · 连续 N 天" (Caption)
- 正中央: Display 128rpx 计时器，呼吸动画
- 下半区: 暂停按钮 (Ghost) + 结束按钮 (Primary CTA)，分占左右
- 底部 tab bar 完全隐藏
- 进场动效: 0.32s fade + scale 0.96 → 1
- ⚠️ 注意 `env(safe-area-inset-bottom)` 防 iPhone 底部条遮挡

### 7.3 日历

- 顶部: 月份导航 (上月 / 标题 / 下月)
- 中段: 7×6 grid 热力图，6 档绿色深浅
- 图例: 横排紧凑 "少 □□□▣▣▣ 多"
- 下段 (仅在选中某天后渲染): 当日 sessions 列表
- 当月无数据: 上方一个 friendly empty card "本月还没有打卡记录"

### 7.4 我的 (profile)

1. 用户卡 — 头像 (圆形，无角标) + 昵称
2. 2×2 stat grid — 累计学习 / 完成打卡 / 最长连签 / 单日最长
3. 菜单列表 4 项 — 成就 / 六科进度 / 学习设置 / 学习日报
4. 版本号 footer

绝对不放: 每日金句、24h 学习时段图、社交分享 (已下沉到设置内)

### 7.5 完成打卡

只要 3 个区域:
1. Hero: "这次专注了 X 分钟"
2. 表单: 照片 (可选 0-3 张) + 一句话总结 (可选) + 科目 chip + 标签 chip
3. 完成打卡 CTA (永远可点)

成就解锁瞬间 → 全屏覆盖 modal，含品种 SVG + 名字 + 稀有度 pill + 解锁条件。

### 7.6 成就墙

- Hero: 大数字 "6 / 11" + 最近解锁品种名
- 按稀有度分组的 2 列网格
- 每个 tile: 品种 SVG (120rpx 见方) + 名字 + 解锁条件 + 进度条 (未解锁时)
- 锁定状态: grayscale 0.85 + opacity 0.55
- 已解锁 epic/legendary: 右上角小星/王冠角标

## 8. 微交互规范

| 交互 | 反馈 |
|---|---|
| 任何可点击元素 | scale 0.97 + 100ms |
| Chip 切换 | 220ms 颜色 + 微缩 |
| 表单提交成功 | 1.2s "已记录" toast |
| 表单提交失败 | toast 显示错误的截断片段，便于截图反馈 |
| 滑动到底部 | 不弹"没有更多了"，静默 |
| 拉刷新 | iOS 原生样式 |
| 长按 | 仅用于图片预览，其余功能不依赖长按 |

## 9. 动画规范

- 页面进入: 自上而下错落 fade-up，每块延迟 +50ms
- Modal 进入: 320ms scale 0.96→1 + fade
- 数字变化: 渐变 220ms，避免硬切
- 进度条: 600ms ease-out 增长 (一次性)
- focus mode 计时器: 4s 呼吸
- 成就解锁: 600ms 整体进场 (大字弹跳 + halo 脉冲 2s ease-out 无限循环)

**绝对不要**: 抖动、震动、wiggle、bounce 弹簧曲线、循环 GIF。

## 10. 参考产品 (借鉴风格而非抄)

- **Things 3** — 极简密度，温和灰阶
- **Bear** — 优雅字体阶梯，留白节奏
- **Linear** — 沉稳深色模式、精准动效
- **Streaks** — 圆形进度仪式感
- **Apple Health 趋势卡片** — 数字 hero + 微缩说明的组合

**不要参考**: Forest (太游戏化)、番茄 ToDo (太密集)、Notion (太工程感)

## 11. 输出要求

请按以下顺序产出:

1. **设计 tokens** (JSON 或 CSS variables) — 颜色、字号、间距、圆角、阴影、曲线
2. **核心组件库** (Figma 或 HTML 渲染) — 9 个组件全状态展示
3. **5 个核心页面 mockup**
   - 首页 idle
   - 首页 focus mode (深色)
   - 完成打卡 + 成就解锁 modal
   - 我的 + 设置
   - 日历 + 单日详情
4. **3-5 个关键交互的动画 spec** (CSS keyframes / Lottie / 文字说明)
5. **Dark mode 适配版** (至少首页 idle 和 focus mode 两屏)

每个 mockup 都要标注:
- 字号 / 字重
- 颜色 token
- 间距值
- 圆角值

## 12. 约束

- 微信小程序 WXSS 不支持 CSS custom properties 在 `:root` 全局，必须每个组件局部声明
- 不支持 `backdrop-filter` (部分老版本)、`gap` 在 grid 内 (用 padding 模拟)
- 必须考虑 750rpx 设计宽度
- 必须考虑 OLED 续航 (大面积纯白慎用)
- 必须支持中文字符 (英文为辅)
- 必须照顾 `@media (prefers-reduced-motion: reduce)` 用户

## 13. 验收标准

设计稿应满足:

- [ ] 任意一屏，描述功能不超过 3 个词
- [ ] 不出现 emoji 装饰 (功能性 emoji 可保留，如成就角标)
- [ ] 不出现感叹号或鸡汤文案
- [ ] 数字字符 (0-9) 全部 tabular-nums 等宽
- [ ] focus mode 单屏可被一眼看清，无任何二级元素
- [ ] 深色模式覆盖至少 80% 页面
- [ ] 所有按钮在 360px (iPhone SE) 上文字不截断、不溢出
- [ ] 所有计时数字在 750rpx 宽度上"00:00:00" 不溢出

## 使用建议

1. **不要一次性扔进 Claude Design**。先扔 §1-§5 (产品定位 + 视觉系统) 让它出一版 design tokens，确认风格对路再继续。
2. **设计稿出来后，用 §13 的 checklist 验收**。任何一条不过就让它改。
3. **focus mode 深色 + idle 浅色** 这个混合策略要让 AI 工具明确知道，不然它默认会全浅色或全深色。
4. 如果工具支持，把 `docs/ui-review-checklist.md` 也一起喂进去，它会避免重蹈已经踩过的 9 类坑 (按钮文字偏、tab bar 重叠、emoji baseline 漂移、圆形 mask 卡角标等)。
5. 出图后建议把设计稿截图沿当前代码结构估算迁移工作量，再决定全量替换还是渐进迁移。

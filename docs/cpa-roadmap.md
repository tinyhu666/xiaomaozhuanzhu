# 小猫专注 — CPA 备考路线图（聚焦：记录时长 + 复盘）

> 取代旧的 `v0.31-plus-roadmap.md`（那份写于 codex 重设计之前，范围偏宽）。
> 本路线图把范围**收死在 CPA 备考工具**，并以用户明确的两个核心为轴：
> **① 记录学习时长　② 复盘**。其它一律降级或冻结。

写于 v0.32.7 之后。

---

## 0. 现状基线（已经有的）

**记录**：每段专注存 `时长 / 科目(会计·审计·税法·财管·经济法·战略 六选一) /
标签(顺利·卡住·高效·复习·刷题·新课) / 一句话总结(≤80字) / 照片(≤3)`。
计时支持 开始→暂停→继续→结束→提交。

**复盘**：日历热力图 + 日详情；周小结弹窗（总时长/次数/7天柱状/Top2科目/
一句建议，周日周一自动弹）；月小结弹窗；六科进度 vs 目标；累计统计
（累计时长/打卡次数/最长连签/单日最长）。

**核心短板**：记录是**科目级**的，复盘是**被动统计弹窗**。多科备考最需要的
那个判断——「我有没有把时间花在该花的科目上」——目前没有任何地方回答。

---

## 1. 设计原则（不可违背）

- **只做 CPA 备考**。不做社交、动态、排行榜、广场。
- **记录 + 复盘 是主线**，游戏化（成就/猫）维持现状、不扩张、不抢入口。
- **低信息密度 + 少操作步骤**（用户反复强调）。每加一个输入都要问"能不能不加"。
- **不新增 tab**（复盘挂在日历/我的下，或作为首页卡片浮出）。
- 不做系统深色模式；专注页深色是刻意的感官切换，保留。
- 记录科目只在完成页选，绝不走 URL 传参（v0.21.2 故障教训）。

---

## 2. 主线 A：记录学习时长（让记录完整、准确、低摩擦）

### A1. 补录（手动补一段学习）★ 高价值
**问题**：忘了开计时 / 在纸质书上学了 2 小时 → 这段时间今天彻底丢失。
对一个"记录学习时长"的工具，这是最大的完整性漏洞。
**方案**：首页/日历加「补录」入口 → 小表单：日期 + 时长 + 科目 + 标签
（可选总结）。一步提交。
**服务端**：复用 complete 的存储，加 `manual: true` 标记（或 `/sessions/manual`），
时间直接取用户填的值而非计时。
**摩擦**：一个小表单，可接受——它是把"丢失的时间"找回来。

### A2. 暂停超时 / 挂死自动结束 ★ 准确性
**问题**：暂停后忘了结束 / 关掉小程序 → session 永远挂在"专注中"，
污染记录、第二天打开看到假的"还在专注"。
**方案**：服务端 cron 扫描（已有 20:30 提醒 cron 可复用）：暂停 >30 分钟
或运行 >8 小时的 session → 自动按**纯运行时长**（不含暂停）结束。下次打开
toast 告知「上次有段没结束的专注，已按 N 分钟自动记录」。
**服务端改动**：需提醒用户在云托管重新部署。

### A3. 科目子主题/章节（可选）△ 需你拍板（摩擦权衡）
**问题**：现在只到科目级。"会计 90 分钟"无法支撑复盘里"哪些章节弱/没碰"。
**方案**：完成页科目下方加一个**可选** topic 输入，带"最近主题"自动补全
（从历史 session 提取）。把 `会计` 变成 `会计·金融资产`。
**权衡**：这是"记录粒度"和"少操作步骤"的直接冲突。做了，复盘能到章节级
（A3 → B 的 topic 覆盖图、章节弱项）；不做，复盘只能到科目级。
**建议**：默认**不做**，除非你确认愿意为章节级复盘多点一下。可先做成
"完全可选、不填零成本"，观察使用率再决定是否强化。

---

## 3. 主线 B：复盘（把被动统计变成可行动的回顾）★★ 最高优先

### B1. 科目 × 考期 平衡复盘 ★★ 最高价值，建议先做
**这是整个产品最缺、对多科备考最致命的一个洞察。**
把每科的「已投入时长 / 目标」叉乘「距该科考期天数」，算出**失衡/欠债**排序：

> 审计 · 距考 88 天 · 已学 3 分钟 → 「严重落后，建议本周优先」
> 会计 · 距考 88 天 · 已学 103 小时 → 「投入充足」

一个**专门的复盘视图**（不是弹窗），按"欠债风险"给六科排序，一眼看出
"我把时间花偏了"。
**数据**：完全复用现有的 科目目标 + 考试日期 + 累计时长，**零 schema 改动**。
**这是 v0.33 的首发功能。**

### B2. 周复盘仪式（把周小结弹窗升级成回顾）★★
现在的周小结是一次性统计弹窗，看完即走。升级成**可行动的回顾**：
- 时间去哪了（已有：7 天柱状 + Top 科目）
- ＋ B1 的科目失衡提醒
- ＋ 状态聚合（见 B3）：「本周'卡住'X 小时，集中在财管」
- ＋ 一个**复盘输入框**：「这周复盘：…」——**存下来、可回看**
把"统计"变成"我这周做得怎样、下周调整什么"的仪式。

### B3. 状态/效率聚合 ★ 让已有标签真正有用
顺利/卡住/高效 这些标签现在只存不算。按科目/按周聚合：
> 财管 60% 的时间标记为"卡住" → 这科要换方法或加时间。
让用户记的标签产生回报，也反哺 B1 的优先级判断。

### B4. 复盘记录库 ★ 让复盘复利
B2 写的周复盘累积成一个**可回看的列表**。三周前对自己说的话、定的调整，
现在能翻出来看做到没有。复盘只有能回看才有复利。

---

## 4. 主线 C：支撑（低优先，做完 A/B 再说）

- **C1 趋势**：随考期临近，学习时长是否在爬升（备考强度曲线）。
- **C2 章节覆盖图**（依赖 A3）：哪些章节碰过/没碰过——考前查漏。
- **C3 复盘导出**：现有日报海报是按天；加一个按周/复盘的可分享卡片。

## 5. 建议排期

| 版本 | 内容 | 依赖 | 服务端改动 |
|---|---|---|---|
| **v0.33** | **B1 科目×考期平衡复盘**（首发，零 schema） | 现有数据 | 否 |
| v0.34 | A1 补录 + A2 暂停超时自动结束 | — | 是（部署提醒） |
| v0.35 | B2 周复盘仪式 + B4 复盘记录库 | A 完成更佳 | 是（存复盘文本） |
| v0.36 | B3 状态/效率聚合 | — | 可能 |
| v0.37+ | A3 章节粒度 + C2 覆盖图（需你拍板摩擦） | A3 | 是 |
| 随手 | C1 趋势 / C3 复盘导出 | — | 否 |

**首发强烈建议 v0.33（B1）**：价值最高、零数据迁移、纯前端 + 现有接口，
能立刻让"复盘"这条线从 0 到 1，且不碰高风险的 schema/服务端。

## 6. 明确不做（防止再被拉偏）

- ❌ 游戏化扩张（成就/猫维持现状，不加第二个入口、不抢主线）
- ❌ 新 tab（复盘不单开 tab）
- ❌ 系统深色模式（已在 v0.31.x 移除）
- ❌ 番茄钟参数回归（已简化掉）
- ❌ 任何"为了好看"增加的信息密度
